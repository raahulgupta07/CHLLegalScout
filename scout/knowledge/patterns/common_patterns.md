# Common Search Patterns

## Finding Policies and Handbooks

**Best approach:**
1. Start with `company-docs/policies/`
2. Search for the policy name or related terms
3. Read the full document, not just snippets

**Known locations:**
- Employee Handbook: `company-docs/policies/employee-handbook.md`
- PTO Policy: In the Employee Handbook, Section 4
- Data Retention: `company-docs/policies/data-retention.md`
- Security Policy: `company-docs/policies/security-policy.md`

**Gotcha:** PTO info is in the Employee Handbook, not a standalone doc.

---

## Finding Runbooks and Procedures

**Best approach:**
1. Start with `engineering-docs/runbooks/`
2. Search for the procedure name
3. Read the full runbook

**Known locations:**
- Deployment: `engineering-docs/runbooks/deployment.md`
- Incident Response: `engineering-docs/runbooks/incident-response.md`
- On-Call Guide: `engineering-docs/runbooks/oncall-guide.md`

---

## Finding OKRs and Planning Docs

**Best approach:**
1. Start with `company-docs/planning/`
2. Search for the quarter (Q1, Q2, Q3, Q4) and year
3. Read the full OKR document

**Known locations:**
- Q4 2024 OKRs: `company-docs/planning/q4-2024-okrs.md`
- Company Strategy: `company-docs/planning/2024-strategy.md`

---

## Finding Technical Documentation

**Best approach:**
1. Start with `engineering-docs/architecture/`
2. For RFCs, check `engineering-docs/rfcs/`
3. For broader topics, search across all directories

**Known locations:**
- System Overview: `engineering-docs/architecture/system-overview.md`
- API Design: `engineering-docs/architecture/api-design.md`
- RFCs: `engineering-docs/rfcs/`

---

## Finding Reports and Metrics

**Best approach:**
1. Start with `data-exports/reports/`
2. Search for the metric name or time period
3. Check company-docs/planning/ for goal-related metrics

**Known locations:**
- Q4 2024 Metrics: `data-exports/reports/q4-2024-metrics.md`

---

## Search Strategy

When information might be anywhere:

1. **Identify the information type**
   - Policies/formal docs → `company-docs/`
   - Technical docs/runbooks → `engineering-docs/`
   - Data/reports → `data-exports/`

2. **Search primary directory first**

3. **Note timestamps** — Newer info may supersede older

4. **Try broader searches** — If specific terms fail, use synonyms

5. **Save what you learn** — If the location was surprising, save it

---

## Handling "Not Found" Results

If search returns nothing:

1. **Try synonyms** — "PTO" vs "vacation" vs "time off"
2. **Broaden the search** — Remove specific terms
3. **Check parent documents** — Info might be in a section of a larger doc
4. **Search across all directories** — Don't limit to one folder
5. **Ask for clarification** — User might know the exact location
